Defining activities

An activity is a rule that matches one or more running apps. When a matching app is in focus, Ludex counts the time against that activity.

Ludex ships with a library of common games and apps. To see them, click Ludex → Advanced → Install standard activities in your dashboard.

Finding an app's process name

To define a custom activity, you need the app's process name as the operating system sees it. Run this on the child's computer while the app is open: 

ludex --detect-app

This prints a list of running processes. Find your app and note the name shown — that's what goes in the definition.

Activity definition format

Each activity is a JSON definition stored in the activity_types tab of your sheet:

{
  "match": {
    "process_name": ["minecraft", "java"]
  },
  "limits": {
    "daily_max_minutes": 60,
    "warn_before_minutes": 10
  }
}
  • process_name — list of process names that count as this activity (case-insensitive)
  • daily_max_minutes — daily time limit; 0 means no limit
  • warn_before_minutes — how many minutes before the limit to warn the child

Adding an activity via the dashboard

  1. Open the activity_types tab in your Google Sheet
  2. Add a new row: fill in activity_id (a short unique slug, e.g. minecraft), name (the friendly display name), and the JSON definition above
  3. Set enabled to TRUE
  4. The agent picks up the new activity on its next sync (within a few minutes)
✅ Use the name column for what your child sees in warnings, e.g. "Minecraft" rather than "minecraft".

Time limits & warnings

Limits are set per activity inside the JSON definition (see above). You can also edit them through the dashboard:

  1. Click Ludex → Edit activity limits
  2. Adjust the daily max and warning threshold for each activity
  3. Save — the agent picks up changes on its next sync

When a child reaches the warning threshold, a notification appears on their screen. When they hit the daily limit, another notification fires. The agent does not enforce limits automatically — enforcement is a manual decision you make as a parent.

ℹ️ The dashboard highlights rows in red when a limit is exceeded and amber when a child is within the warning window.

Sending commands

Commands let you act on a child's computer from the dashboard without being in the room. Click Ludex → Send a command and choose:

  • Stop activity — kills a specific running app by name
  • Shutdown — shuts down the computer (requires the OS to allow it for the user)

Commands are queued and picked up on the agent's next sync cycle (within a few minutes). The dashboard shows whether the command was executed or failed.

⚠️ The agent only kills processes owned by the logged-in user. It cannot force-quit system processes or apps running as another user.

Friendly names

By default, each child appears in the dashboard as their system username and hostname. To give them a friendly name:

  1. Click Ludex → Edit people — the people tab opens
  2. Edit the name column next to the user you want to rename
  3. Optionally add an email address to receive limit-exceeded alerts for that user

Updating the dashboard script

When a new version of Ludex is released:

  1. Replace the script files in Apps Script with the new versions
  2. In the script editor, click Deploy → Manage deployments
  3. Click the edit (pencil) icon → set version to New version → Deploy

The URL stays the same — you don't need to update the agent.

To update the agent on a child's computer, download the new binary from the releases page and run it. It will replace the existing installation.

Uninstalling

Remove the agent from a computer

On the child's computer, open a terminal and run:

ludex uninstall

This stops the background service and removes it. The binary itself is not deleted — you can move it to the trash manually.

Remove a computer from the dashboard

The users tab in your Google Sheet lists every computer that has ever checked in. To remove an entry, delete the row. Historic activity data in the activity_log tab is unaffected.

Remove the dashboard entirely

Delete the Google Sheet and the Apps Script project — nothing else to clean up. All data is in your Google account and goes with it.
